# Delivery Management System - Master Database Schema V2

This schema is optimized for **PostgreSQL** and **Laravel Eloquent**, utilizing PostGIS for spatial data, strict decimal typing for financial accuracy, and **ULID** for secure, sortable primary keys across all tables.

## 🏢 Domain 0: Multi-Tenancy & Financial Config

### 1. `companies`
The root tenant. All operational data strictly isolates by this ID using Laravel Global Scopes.
| Column | Type | Constraints / Notes |
| :--- | :--- | :--- |
| `id` | ULID | Primary Key |
| `name` | String | |
| `telegram_user_id`| String | Nullable, Unique within company (For Telegram Bot alerts/auth) |
| `tax_id` | String | Nullable |
| `base_currency` | String | Default: 'USD' |
| `timestamps` | Timestamp | `created_at`, `updated_at` |

### 2. `exchange_rates`
Master table for daily/hourly rate tracking.
| Column | Type | Constraints / Notes |
| :--- | :--- | :--- |
| `id` | ULID | Primary Key |
| `company_id` | ULID | Foreign Key (`companies.id`), Indexed |
| `from_currency` | String | e.g., 'USD' |
| `to_currency` | String | e.g., 'KHR' |
| `rate` | Decimal(15,6) | High precision for accurate conversion |
| `effective_date` | Timestamp | |
